# AI 物語生成エンジン 技術設計書 Version 1.0

## 1. システム概要

本システムは、LINE 風チャット UI を入力面に持ち、会話の流れから長編物語を進行させる生成エンジンである。  
翻訳モードとは別系統で動作し、物語の継続性、キャラクターの一貫性、世界観の整合性を維持することを主目的とする。

### 目的
- 会話入力を物語進行に変換する
- 章、シーン、ビート、行の階層を維持する
- キャラクター記憶と世界観知識を保持する
- 整合性チェックを経由して破綻を抑制する
- Rust + WASM を中核として Web UI に統合する

## 2. 要求仕様

### 機能要件
- 世界観生成
- キャラクター生成
- 章構成生成
- シーン分割
- Beat 生成
- セリフ生成
- 地の文生成
- 整合性チェック
- 記憶更新
- UI 連携

### 非機能要件
- 長編でも破綻しにくい
- ローカル実行可能
- モジュール境界が明確
- LLM 呼び出しは小刻み
- 永続化が可能

## 3. 生成モデル要件

### 生成モデル
物語本文の生成には長文保持と日本語品質を重視した LLM を利用する。  
プロンプトに依存するため、各生成ステージで出力形式を固定する。

### 整合性モデル
長文の整合性確認、キャラクターの口調確認、伏線の取りこぼし確認に別経路を用意する。  
生成と検査を同一処理にしない。

## 4. 翻訳方針

本仕様では、翻訳は主機能ではない。  
物語生成において翻訳が必要な場合は、生成モデル内部または別の補助処理として扱い、独立した翻訳モデルの責務にしない。

## 5. アーキテクチャ

### 構成
- UI 層: LINE 風チャット、編集パネル、状態表示
- 生成層: ストーリー構成、本文、整合性確認
- 記憶層: キャラ記憶、世界観知識、会話履歴
- 永続化層: IndexedDB または SQLite
- 実行層: Rust + WASM

### 処理フロー
1. テーマ入力
2. 世界観初期化
3. キャラクター初期化
4. 章構成生成
5. シーン生成
6. Beat 生成
7. 本文生成
8. 整合性チェック
9. 記憶更新
10. UI に反映

## 6. データモデル

### Story
```rust
struct Story {
    id: StoryId,
    title: String,
    theme: String,
    world_id: WorldId,
    characters: Vec<CharacterId>,
    chapters: Vec<ChapterId>,
}
```

### Chapter
```rust
struct Chapter {
    id: ChapterId,
    index: u32,
    title: String,
    summary: String,
    scenes: Vec<SceneId>,
}
```

### Scene
```rust
struct Scene {
    id: SceneId,
    index: u32,
    pov: CharacterId,
    location: LocationId,
    summary: String,
    beats: Vec<BeatId>,
}
```

### Beat
```rust
struct Beat {
    id: BeatId,
    index: u32,
    description: String,
    lines: Vec<Line>,
}
```

### Line
```rust
enum Line {
    Dialogue(DialogueLine),
    Narration(NarrationLine),
}
```

## 9. パイプライン

### Stage 1: 企画生成
テーマ、トーン、ジャンル、登場人物を生成する。

### Stage 2: 章構成生成
物語全体の流れを章単位で切る。

### Stage 3: シーン分割
章をシーンに分け、視点、場所、目的を付与する。

### Stage 4: Beat 生成
シーンを行動単位に分割する。

### Stage 5: 本文生成
会話と地の文を生成する。

### Stage 6: 整合性チェック
性格、口調、ルール、死亡キャラ、伏線回収を確認する。

### Stage 7: 記憶更新
記憶と世界状態を更新する。

## 10. LLM 呼び出し設計

### 原則
- 一度に大きく作らない
- 小さい単位で分割する
- 各ステージの出力形式を固定する

### 呼び出しの粒度
- 企画: 少数回
- 章: 章数ごと
- シーン: シーンごと
- Beat: Beat ごと
- 整合性: 章またはシーン単位

## 11. プロンプト設計

### 基本方針
- 役割を明示する
- 制約を明示する
- 出力形式を固定する
- キャラ設定を毎回渡す

### 出力形式
JSON か固定テキストのいずれかに統一する。  
UI に戻す部分は機械可読性を優先する。

## 12. 整合性チェック

### チェック項目
- キャラクターの性格
- 口調の揺れ
- 世界観ルール違反
- 死亡キャラの再登場
- 伏線の未回収
- 地理的矛盾

### 返却形式
```json
{
  "issues": [],
  "suggestions": []
}
```

## 13. 記憶更新アルゴリズム

1. Beat を解析する
2. 感情変化を抽出する
3. 関係性変化を抽出する
4. 世界観変化を抽出する
5. `MemoryEntry` を追加する
